I want to begin by saying the TFs – specifically Sarah and Francesco – held down the fort and made this course bearable. I also want to add that Michael is a fun and engaging person and professor. I did have some fun such as the lava cake lab but this course has many pitfalls:
1) There is little to no organization in this course. Each week you kind of learn a new random model and then move on – I feel like I didn't learn anything or come out of this course with any tangible skills. Moreover, (and again hopefully this gets better) in the first week when the problem set was released the TFs did not have the solutions, this got better but that was shocking and unacceptable. Since section (and lecture) had optional attendance, no one attended anything and the class had little to no community and I didn't really meet anyone new in the course. And because everything was optional, the TFs (who were trying their best) had no centralized guidance on what to teach in sections leading to huge variation in both material and quality being taught in section (that said it's not their fault because I can imagine it is horrible to prep and teach a section when literally no one shows up). Another example is how we had everyone present a 45 second slide on their project proposal in our big class – not only was this chaos and did not work it was completely unproductive since we basically received no feedback (because it would have naturally taken too long)
2) The mini–projects and individual project at the end were interesting because they were problems we came up with but again because there was little bounds on what the project was supposed to look like people did the most random things. There needs to be much more thought put into these projects to make them productive.
Lastly some advice if you have to take this course you should be able to code – not understanding code – this is not a course which teaches coding but uses a lot of code so it's important to have a good foundation.
