This was my least favorite class here at Harvard, and especially within the stats department. The class is extremely focused on methodology and detailsif you want to learn how to do regressions and make plots in R, or become an expert in interpreting model coefficients, or know how to run significance tests without understanding why they work, this would be an excellent choice. However, there is little explanation given for the intuition behind much of anything. As a result, this class feels more like an instructions manual for running R regressions than a real stats class, and I think many of my peers share this sentiment.

For example, GLMs require the choice of a link function to introduce nonlinearity into the model. Intuitively, it seems that the validity of your regressions are highly dependent on what link function you choose. I still have no idea how, though, because the way link functions are taught in class is just as a bullet list of criteria which has to be fulfilled. No concept of how two "valid" functions affect the model outcome differently, or why we can still perform regressions when the data does not perfectly fit the link function. This is only one example of countless that appear throughout the class (the explanation of deviance and likelihoods is also super annoying). I understand that among all stat classes, this lands on the more practical, data–sciencey side, but I wish there were more time put into big ideas, motivations, and getting a slightly deeper understanding of the many tools you learn to use.

Grading is based on a checklist–style rubric which becomes extremely repetitive throughout the semester. Import the same packages, make the same regressions, make the same plots, write two paragraphs for each part of a question because it asked you to "interpret the coefficients." There is specific language you have to use (ie. holding all other variables constant) or else points are taken off. Be super explicit about every statement or points are taken off. Past the first, psets are just busywork most of the time, though they are biweekly and not too work intensive. Some required knowledge for tests are just memorization exercises. Also, coding in R is a must, which sucks (super ugly language).

I really wish Stat 244 were still offered, because I'm sure it would be a much more interesting class than this. Out of the three core stat requirement classes, this one is easily the most boring by far, though I don't know how it compares to 139. If you are more interested in the math/stats side of things, I'd avoid the 139/149 series.
