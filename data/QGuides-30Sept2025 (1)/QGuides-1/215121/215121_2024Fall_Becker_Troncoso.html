Not much work outside of the milestones (the HWs are really quick except for HW2 which is still manageable), but the last 3 definitely are a lot of work and will require long stretches of locking in. Make sure you have a group that can split up the work evenly and to choose a project that isn't too hard. If it requires you having to build and train a model from scratch, I wouldn't do it unless you previously had a set plan of how to do it. The class is about productionizing AI and not training a model. Your ML pipeline is just one bit of your project. You still have to build a front–end for it (which I would make cool because that sets you apart from the rest), deploying it, and finally scaling it

Practically, this was one of the most helpful ML classes I've taken – I finally know my way around Docker, cloud services (GCP in this case), version control, and Kubernetes at the end – all technologies that I'm sure will be helpful in industry and for building my own future projects. This is the best (albeit only) MLOps class that Harvard has to offer. Rather than learning about writing optimized code or about the ins and outs of ML architectures, you learn how to deploy and productionize models with user–facing applications. This class is incredibly helpful for preparing students to go into industry within CS/ML/data science, not only because you learn best practices for MLOps (e.g., storing data, serverless training, creating API services, creating a front–end, scaling), but also because you will be working with a team on your project, resembling a SWE/ML team in industry.

Good and useful materials if you want to become SWE/MLE. Pavlos's teaching is ok, but Ignacio's teaching is as bad as before and he doesn't quite understand the materials. If you take this class, I would recommend you go to Pavlos's and TFs' OHs.
AC215, it's been a wild ride. I believe that this is a young course, so the staff are still figuring things out. If you have taken 109A and B, you know that Pavlos is a master of his craft when it comes to teaching data science. He really "gets it" when it comes to teaching what's needed and giving you the resources to teach yourself everything else.

Besides Pavlos, the lecturers were extremely dry, so much so that the tutorials took a few tries to understand. One result of this was that work piled up at the end of the semester because frankly, we didn't understand what were doing at the start of the semester. I hope that in the future, there are more (but smaller) assignments to allow students to learn lot at the beginning of the course before they need to apply it at the end. That would make it feel more balanced.

The other half of that story, of course, is that you need to stay disciplined in this class. If you pick a project that goes beyond the basic RAG / chatbot template that the course promotes, be prepared for substantial coding on your own. There is a large amount of coding, debugging, and asking for help that is only possible if you leave yourself time to work on your project. It is very possibly to breeze through this course by picking a project that is close to the tutorials, but I think it is more rewarding to go off the beaten path –– but be prepared for work.

The main takeaways are:
– this is a young class without much structure, so you will have to teach yourself things
– the lectures, besides Pavlos, are dry but content–rich
– you need to be disciplined with the project
